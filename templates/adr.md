# Architecture Decision Record — Template

One file per consequential decision, saved at `docs/08-architecture/adr/ADR-XXX.md` in the target project (see `rules/document-locations.md`). Created whenever a playbook (typically 08-Architecture, but any phase may produce one for a decision with a high cost of reversal — see `rules/confirmation-protocol.md`'s always-strict categories) confirms a decision worth recording for future reference.

## Structure

```yaml
---
id: ADR-001
title: "<short, specific title — e.g. 'Use PostgreSQL as the primary datastore'>"
status: "accepted"
# "proposed" | "accepted" | "superseded" | "deprecated"
date: "2026-07-13"
traces_to: []
# REQ-XXX (non-functional requirements this decision addresses) and/or
# ARCH-XXX (the component/decision this ADR documents), per rules/traceability-rules.md
superseded_by: null
# ADR-XXX, if status is "superseded" — a superseded ADR is never deleted,
# only marked, per rules/id-conventions.md
---
```

```markdown
## Context
<What situation/problem/requirement prompted this decision? Reference the
REQ-XXX or discussion that led here — do not restate the whole requirement,
just enough context for the decision below to make sense on its own.>

## Decision
<The decision, stated as a single clear sentence. Not a discussion — the
conclusion.>

## Alternatives considered
<Every real alternative that was on the table, one bullet each, with the
one-line reason it was rejected. If there was truly only one option, say so
explicitly — do not fabricate alternatives to fill this section.>

## Consequences
<What this decision makes easier, what it makes harder, and what it
forecloses. Both positive and negative — a decision with no stated
downside was not actually analyzed.>
```

## Notes for whoever fills this in

- The `Context` and `Decision` sections must reflect what the user actually confirmed (`rules/confirmation-protocol.md`) — never a rationale the AI constructed after the fact to justify a choice the user didn't explicitly make.
- If a later Change Request (`rules/change-management.md`) changes this decision, do not edit this file's `Decision` in place — create a new ADR, mark this one `status: superseded`, and set `superseded_by`.
