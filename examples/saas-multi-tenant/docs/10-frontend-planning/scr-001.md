# SCR-001 — Task Board
*Traces to: UC-002 · Author: Priya*

**Description**: Where a tenant member reviews and filters their project's tasks, and starts the create-task flow.

**Composition**: FilterBar → TaskList → TaskRow → [inline status control]

**Calls**: API-002

**Permission**: Any authenticated member of the tenant, scoped to their own tenant's tasks (tenant isolation per REQ-003/ARCH-002).

**States**
| State | What the user sees |
|---|---|
| Loading | Same as General conventions |
| Empty | "No tasks yet — create one to get started," with the New Task action visible |
| Error | Same as General conventions |

**Responsive behavior**
Single-column task list below the mobile breakpoint (not a primary use case for MVP, but not broken); full table layout at tablet and above.

**Analytics events**
`task_board_viewed` (on load, with project id and applied filters), `task_filter_applied` (on filter change).
