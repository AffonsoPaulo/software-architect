# docs/

This skill's own usage documentation — for people using or maintaining the Skill itself. Not to be confused with the documents the Skill *generates* inside a user's project (those live at `/docs/<phase>/` in the target project's repository, per `rules/document-locations.md`).

- `how-it-works.md` — the full phase flow, the orchestrator, `project-state.md`, confirmation modes, document format, documentation depth, traceability
- `faq.md` — anticipated questions (skipping phases, switching confirmation mode, documentation depth, incremental cycles, unknown answers, brownfield support)
- `troubleshooting.md` — common failure scenarios and how to recover
