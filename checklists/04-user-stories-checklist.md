# User Stories Checklist

- [ ] Every `US-XXX` ID is unique and correctly formatted
- [ ] Every `US-XXX` has a non-empty `traces_to` pointing to an existing `REQ-XXX`
- [ ] Every functional `REQ-XXX` is covered by at least one `US-XXX`, unless documented as a non-functional exception
- [ ] Each story is phrased from the actor's perspective, not restated implementation detail
- [ ] The persona is specific, not a generic placeholder
- [ ] The benefit is genuine value, not a restatement of the action
- [ ] INVEST holds well enough to proceed — Independent and Testable in particular
- [ ] Every story was explicitly confirmed by the user
